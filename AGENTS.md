# AGENTS.md（步步兽项目规则）

本文件补充 `D:\repo\AGENTS.md`。发生冲突时，安全边界与用户最新明确要求优先。

## 当前阶段

- 当前处于规格设计阶段。
- 未获得用户对 `docs/spec-approval.md` 的确认前，不写业务代码、不安装依赖。
- 当前开发目标只能是 V0.1；不得提前实现 Hermes、Obsidian 写入、电脑行为监控或复杂商城。

## 开发方法

1. 开始任务前，只读取与当前任务直接相关的文档和代码。
2. 先说明当前 Milestone、技术选择及选择原因，再修改代码。
3. 每次只完成一个可验收的小功能，尽量控制在 1—5 个文件。
4. 每完成一个功能立即运行对应测试；测试失败不得宣称完成。
5. 不自行更换技术栈，不进行无关重构，不覆盖用户已有改动。

## Subagents 与 Token 规则

- 默认不调用 subagents。
- 只有用户明确要求，或任务能拆成彼此独立、确实可并行的多个大型部分时，才允许调用。
- 调用前必须写明目标、输入边界和输出格式；默认最多 1 个，用户明确要求时最多 2 个。
- 单文件修改、Bug 修复、普通组件、代码解释、目录检查不得调用 subagents。
- 不全量读取仓库，不重复输出完整 PRD，不为简单问题加载历史长上下文。
- 优先搜索后定点读取；输出只保留结论、变更、测试和必要说明。

## 技术边界

- 桌面外壳：Electron。原因是 V0.1 需要 Windows 透明、无边框、置顶、托盘与本地能力。
- UI：React + TypeScript + Tailwind CSS。原因是组件化、类型安全，并符合用户熟悉的前端路线。
- 构建：Vite。原因是桌面单页 UI 不需要 Next.js 的服务端渲染，Vite 更轻、更适合 Electron 开发进程。
- 数据：SQLite。原因是单用户、本地优先、无需数据库服务。
- V0.1 不引入 ORM。数据访问先使用小型 repository 层；只有 Schema 复杂度明显增加时再评估 Drizzle。
- API Key 只放 `.env`，仓库只提交 `.env.example`。

## 产品边界

- 不通过扣血、宠物生病或资源清零惩罚用户。
- 延期只触发减负建议，不制造内疚。
- 默认主动等级为 Level 3；检测到高压力或连续延期时降为 Level 2。
- AI 建议不能直接创建大量任务、修改 Obsidian 或执行系统操作，必须经过确认。

## Git 规则

- 提交前检查 `git diff` 和测试结果。
- Commit 使用中文：`feat: 添加xxx功能`、`fix: 修复xxx问题`、`docs: 完善xxx文档`。
- 每个功能一个小提交，并将 commit ID、日期、功能说明写入 `COMMIT-LOG.md`。
- 禁止提交 `.env`、`node_modules`、构建产物和本地数据库。

## 每次交付格式

1. 当前任务
2. 技术选择及原因
3. 修改内容与文件
4. 测试结果
5. 尚未完成或下一步
